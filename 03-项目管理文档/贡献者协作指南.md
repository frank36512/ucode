# 贡献者协作指南

欢迎加入 **优码（ucode）** 开源社区！我们致力于打造一个去中心化、全民共建的优惠码平台。无论你是技术大牛还是普通用户，都可以通过以下方式参与贡献。

## 🌟 核心贡献方向

1.  **商家洽谈**（⭐️ 核心）：对接本地/线上商家，确认合作意向并申请 ucode。
2.  **信息维护**：整理商家/ucode 信息，更新优惠状态（生效/失效）。
3.  **内容核验**：核验 ucode 有效性、商家优惠兑现情况，举报虚假信息。
4.  **规则优化**：提出文档/规则的优化建议。

## 🚀 详细操作流程

### 1. 商家洽谈与 ucode 申请
如果你发现了一家不错的店铺（无论是你常去的餐馆，还是你朋友开的小店），都可以尝试洽谈接入 ucode。

*   **准备**：阅读 `04-资源与工具/商家洽谈话术.md`，了解 ucode 的优势（免费、引流）。
*   **沟通**：与商家确认优惠内容（如“凭 ucode 享 9 折”），确保无套路。
*   **提交**：
    1.  Fork 本仓库。
    2.  复制 `02-协作模板/ucode申请模板.md` 内容。
    3.  在 Issues 中新建 Issue，粘贴内容并填写。
    4.  等待维护者审核（通常 24 小时内）。
    5.  审核通过后，维护者会分配 ucode 编号并合并到核心数据中。

### 2. 信息维护与更新
如果你发现某个 ucode 即将过期，或者商家信息有变：

*   **提交变更**：使用 `02-协作模板/ucode变更_注销模板.md` 提交 Issue 或 Pull Request。
*   **更新文件**：直接修改 `01-优码核心数据/` 下对应的 Markdown 文件，提交 PR。

### 3. 举报与监督
如果你在使用 ucode 时遇到问题：

*   **提交举报**：使用 `02-协作模板/虚假信息举报模板.md` 提交 Issue。
*   **社区核实**：社区成员会进行核实，一旦确认违规，将下架该 ucode 并拉黑商家。

## 🛠️ GitHub 协作规范

*   **Fork & Pull Request**：所有的修改建议请通过 PR 提交。
*   **Commit Message**：请保持清晰，如 `feat: 新增 XX 商家 ucode` 或 `fix: 修正 XX 商家地址`。
*   **Issue**：遇到问题或有新想法，欢迎在 Issues 区讨论。

## 🏆 贡献者激励

所有贡献者的操作均会在 GitHub 提交记录中留痕。
*   **核心贡献者**：长期活跃、贡献突出的成员将受邀成为**项目维护者**，拥有代码合并权限。
*   **荣誉榜单**：你的名字将永久记录在 `README.md` 的贡献者榜单中。
