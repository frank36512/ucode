# ucode 申请模板

> **提交说明**：
> 1. 请复制本模板内容，新建 Issue 或 Pull Request 进行提交。
> 2. 请严格按格式填写，不遗漏必填项，否则将驳回申请。
> 3. 商家洽谈可参考 `04-资源与工具/商家洽谈话术.md`。

## 一、必填项

### 1. 商家基础信息
*   **商家名称**：[准确填写，与门店招牌一致，不可简称]
*   **商家品类**：[如：餐饮-家常菜 / 商超-便利店 / 数码-手机配件 / 线上-电商]
*   **经营地址**（线下商家）：[省+市+区+街道+门牌号，精确到可找到门店]
*   **线上链接**（线上商家）：[如淘宝店铺链接、小程序链接，无则填「无」]
*   **商家联系方式**：[对接人姓名+电话，**仅项目维护者可见，请勿公开在 Issue 中，可通过邮件发送给维护者**]

### 2. ucode 相关信息
*   **申请 ucode 格式建议**：[按品类填写，如餐饮填 UC000X，商超填 US000X，序号可留空]
*   **优惠内容**：[明确无套路，如「满 50 减 20」「凭码享 8 折」「免费送小礼品 1 份」]
*   **优惠有效期**：[YYYY.MM.DD - YYYY.MM.DD，最长不超过 1 年]
*   **核销方式**：[如「线下口头出示 ucode，店员核验」「线上下单时备注 ucode」]

### 3. 申请相关信息
*   **申请人 GitHub 昵称**：[必填]
*   **商家合作证明**：[如洽谈聊天记录截图、商家同意合作的文字确认，请上传至图床并附链接]

---

## 二、可选项

1.  **优惠使用限制**：[如「满减仅限堂食」「每人每天限用 1 次」「新品除外」，无则填「无」]
2.  **商家门店/优惠海报链接**：[上传至免费图床，提升真实性]
3.  **其他补充说明**：[如「商家可提供临时优惠调整，将提前告知维护者」]

---

## 三、申请人承诺

- [ ] 本人已与该商家确认合作意向。
- [ ] 商家自愿认可 ucode 并兑现优惠。
- [ ] 所提供信息真实有效，若存在虚假信息，自愿承担相关责任，退出贡献者行列。
